PRD (Product Requirements Document)

1. 제품 개요

1.1 목적

사용자가 하루 단위로 계획(Plan) 과 실행(Execution) 을
시간 기반 타임라인 UI에서 나란히 비교하여
계획 대비 실행의 차이를 직관적으로 파악할 수 있도록 한다.

⸻

2. 핵심 UX 원칙
	1.	비교 우선
	•	“계획 vs 실행” 차이를 한 눈에 인지
	2.	입력 최소화
	•	실행 기록 시 기존 계획 재사용
	3.	시간 직관성
	•	시간 길이 = 블록 높이
	4.	조작 예측 가능성
	•	드래그/리사이즈 시 시간 변화가 즉시 시각화

⸻

3. 타임라인 레이아웃 정의 (상세)

3.1 컬럼 구조

| 시간 | 계획 (Plan) | 실행 (Execution) |

	•	좌측: 시간 레이블
	•	중앙: 계획
	•	우측: 실행

⸻

3.2 타임라인 그리드 기준 (확정)

시간 레이아웃
	•	1시간 단위 표시
	•	예: 09:00, 10:00, 11:00 …

Plan / Execution 그리드
	•	30분 단위 시각적 분할
	•	1시간 = 2칸

⸻

3.3 픽셀 기준 (권장값, 조정 가능)

기준	값
1시간 높이	120px
30분 단위	60px
10분 스냅	20px
5분 스냅	10px

계산 공식
blockHeight(px) = (endTime - startTime in minutes) × (120px / 60min)

⸻

4. 시간 관리 정책

4.1 시간 단위 정책
	•	내부 데이터: 분 단위
	•	기본 UI 스냅: 10분
	•	설정 옵션: 5분 스냅

⸻

5. 드래그 / 리사이즈 상세 규칙 (신규 추가)

5.1 블록 생성
	•	Plan / Execution 영역 빈 공간 드래그
	•	드래그 시작 지점 → 시작 시간
	•	드래그 종료 지점 → 종료 시간
	•	최소 생성 단위: 스냅 기준(기본 10분)

⸻

5.2 블록 이동 (Drag & Drop)
	•	블록 전체 드래그 시:
	•	시작/종료 시간 동일 간격 유지
	•	이동 중:
	•	스냅 단위로 시간 보정
	•	이동 중 가이드 라인 표시
	•	다른 컬럼 이동:
	•	Plan ↔ Execution 이동 불가
	•	동일 컬럼 내 이동만 허용

⸻

5.3 블록 리사이즈
	•	상단 핸들:
	•	시작 시간 변경
	•	하단 핸들:
	•	종료 시간 변경
	•	최소 길이:
	•	스냅 단위 이상
	•	리사이즈 중:
	•	실시간 시간 툴팁 표시 (예: 17:10 ~ 18:00)

⸻

6. 일정 겹침 처리

6.1 겹침 기준
	•	동일 컬럼 내
	•	시간 범위가 겹치는 경우

6.2 표시 규칙
	•	2개 겹침:
	•	수직 분할
	•	3개 이상:
	•	대표 2개 표시
	•	나머지 +N 표시
	•	+N 클릭 시:
	•	리스트 팝업 또는 모달

⸻

7. 실행 여부 & 달성률

7.1 실행 판단
	•	계획과 동일한 행동을 일부라도 수행 시 “실행”
	•	시간 완전 일치 불필요

7.2 달성률 계산

달성률 = 실행 시간 / 계획 시간 × 100

7.3 시각적 표현
	•	미실행 계획:
	•	흐린 색상 + 점선 테두리
	•	실행된 계획:
	•	달성률 배지 (예: 80%, 120%)

⸻

8. 카드 UI 규칙

8.1 색상
	•	과목 구분용
	•	사용자 지정 가능
	•	상태 표현용 아님

8.2 텍스트
	•	말줄임 처리 (…)
	•	줄바꿈 없음
	•	전체 텍스트는 툴팁/상세뷰 제공

⸻

9. 날짜 네비게이션 & 대표 일정

9.1 날짜 이동
	•	좌/우 스와이프 → 하루 단위 이동

9.2 대표 일정
	•	날짜별 중요 일정 등록
	•	하루 1개 선택
	•	선택 시 날짜 상단에 노출

⸻

10. 사용자 시나리오 (Main Scenario)

시나리오 목적

본 시나리오는 하루 단위 Plan / Execution 타임라인 화면에서 사용자가 계획을 확인하고, 실행을 기록하며, 계획 대비 실행 결과를 비교·조정하는 전체 흐름을 설명한다.

이 시나리오는 다음을 동시에 검증한다.
	•	타임라인 그리드 규칙
	•	드래그 / 리사이즈 인터랙션
	•	계획 ↔ 실행 관계 모델
	•	달성률 및 미실행 표시 UX
	•	날짜 이동 및 대표 일정 노출

⸻

메인 시나리오: “계획한 하루를 실행하고 돌아보는 흐름”

Step 1. 사용자는 오늘 날짜의 타임라인 화면에 진입한다
	•	앱 진입 시 기본으로 오늘 날짜의 Day Timeline 화면이 열린다.
	•	상단에는 다음 정보가 표시된다.
	•	날짜 (예: 2025.09.10)
	•	선택된 대표 일정 (예: “중간고사”)
	•	요약 정보
	•	계획 총 시간
	•	실행 총 시간
	•	화면 중앙에는 타임라인이 표시된다.
	•	좌측: 시간 컬럼 (1시간 단위)
	•	중앙: 계획(Plan)
	•	우측: 실행(Execution)
	•	계획/실행 영역에는 30분 단위 그리드가 보인다.

⸻

Step 2. 사용자는 오늘의 계획을 시간 흐름에 따라 확인한다
	•	사용자는 타임라인을 스크롤하며
	•	언제 무엇을 계획했는지
	•	어느 시간대에 계획이 몰려 있는지
를 직관적으로 파악한다.
	•	계획 블록은:
	•	과목 색상으로 구분되어 있으며
	•	블록 높이는 계획 시간 길이를 그대로 반영한다.
	•	텍스트가 길 경우 말줄임 처리되어 있다.

⸻

Step 3. 사용자는 실행 기록을 추가한다 (계획 기반)
	•	사용자는 실행 컬럼의 특정 시간대(예: 17:00 ~ 18:00)를 길게 드래그한다.
	•	“실행 추가” UI가 나타난다.
	•	실행 생성 시:
	•	오늘 날짜의 계획 목록이 선택 옵션으로 제공된다.
	•	사용자는 기존 계획 중 하나를 선택한다.
	•	과목, 제목, 색상, 기본 시간 정보가 자동으로 채워진다.
	•	사용자는 실제 수행한 시간에 맞게
	•	블록을 리사이즈하여 종료 시간을 조정한다.

⸻

Step 4. 사용자는 계획과 다른 실행도 기록한다
	•	같은 시간대에
	•	계획에는 없던 다른 과목을 실행한 경우,
	•	사용자는 실행 컬럼에서 다시 드래그하여
	•	계획을 선택하지 않고
	•	실행 블록을 직접 생성한다.
	•	이 실행 블록은:
	•	계획과 연결되지 않은 독립 실행으로 저장된다.

⸻

Step 5. 실행 결과가 계획과 자동 비교된다
	•	실행이 하나라도 연결된 계획 블록은 “실행됨” 상태로 판단된다.
	•	계획 시간 대비 실행 시간에 따라:
	•	달성률이 자동 계산된다.
	•	예: 60%, 100%, 130%
	•	달성률은 계획 블록에 배지 형태로 표시된다.
	•	계획했으나 실행되지 않은 블록은:
	•	흐린 색상
	•	점선 테두리
로 시각적으로 구분된다.

⸻

Step 6. 사용자는 겹치는 일정도 자연스럽게 인지한다
	•	동일 시간대에 일정이 2개 겹칠 경우:
	•	블록은 수직 분할되어 나란히 보인다.
	•	3개 이상 겹칠 경우:
	•	대표 2개만 표시
	•	나머지는 +N으로 축약 표시된다.
	•	사용자는 +N을 눌러
	•	해당 시간대의 전체 일정을 확인할 수 있다.

⸻

Step 7. 사용자는 계획 또는 실행을 수정한다
	•	사용자는 계획 블록의 하단을 드래그하여
	•	종료 시간을 30분 늘린다.
	•	블록은:
	•	10분 스냅 단위로 조정된다.
	•	변경 즉시:
	•	계획 총 시간
	•	달성률 계산 결과
가 다시 반영된다.

⸻

Step 8. 사용자는 날짜를 이동하며 흐름을 비교한다
	•	사용자는 화면을 좌우로 스와이프한다.
	•	날짜가 하루 단위로 변경된다.
	•	각 날짜마다:
	•	계획/실행 타임라인
	•	대표 일정
	•	요약 정보
가 함께 갱신된다.

⸻

Step 9. 사용자는 하루를 “돌아본다”
	•	사용자는 오늘 하루를 보며 다음을 한 눈에 확인한다.
	•	계획한 것 중 하지 못한 것
	•	계획보다 더/덜 한 것
	•	계획에 없었지만 실제로 한 것
	•	사용자는 이를 바탕으로
	•	다음 날 계획을 더 현실적으로 조정한다.

⸻

시나리오 핵심 요약 (PRD 관점)

이 시나리오는 다음을 보장한다.
	•	비교 중심 UX: 계획과 실행의 차이가 즉시 보인다.
	•	입력 효율성: 실행 기록 시 계획을 재사용할 수 있다.
	•	시간 직관성: 블록의 크기와 위치가 곧 시간이다.
	•	유연성: 계획과 다른 행동도 자연스럽게 기록된다.
	•	확장성: 이후 주/월 단위 분석으로 확장 가능하다.
